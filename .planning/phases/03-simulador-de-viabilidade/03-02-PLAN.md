# PLAN-02: Implement Simulation Form and Controller Logic

## Objective
Implement the user interface form for simulation inputs and connect it to the ViabilityService.

## Acceptance Criteria
- Form includes fields for: Setor, Fonte atual, Gasto mensal atual, Rota verde, Investimento estimado
- Form validates required fields before enabling calculation
- Form resets to initial state when inputs change significantly
- Controller properly calls ViabilityService with form inputs
- Controller handles and displays calculation results
- Controller handles error states and displays appropriate messages
- Form works with both Firebase and localStorage modes

## Tasks
1. Enhance SimulationController to handle form state and validation
2. Inject ViabilityService into SimulationController
3. Implement form submission handler that calls ViabilityService
4. Create/update simulation view with form fields and result display
5. Implement input validation for numeric fields (positive numbers)
6. Implement dynamic updating of Rota verde options based on Setor/Fonte atual selection
7. Add loading states during calculation
8. Implement result display with formatting (currency, percentages, etc.)
