# PLAN-04: Implement Firebase/LocalStorage Integration for Simulations

## Objective
Implement resilient storage for simulations with Firebase primary and localStorage fallback (DATA-03).

## Acceptance Criteria
- Primary: Attempt to save to Firebase Realtime Database when available
- Fallback: Save to localStorage when Firebase unavailable
- Queue mechanism for pending simulations when offline
- Sync mechanism to push queued simulations when connection restored
- Will not block user flow if storage unavailable
- Each simulation record includes timestamp, inputs, results, and classification
- Implements DATA-03 requirements for Firebase logging
- Storage operations are transparent to the user experience

## Tasks
1. Enhance ViabilityService with saveSimulacao and loadSimulacoes methods
2. Implement Firebase write/read operations with error handling
3. Implement localStorage read/write operations as fallback
4. Create queue mechanism for failed Firebase writes (offline simulations)
5. Implement sync mechanism to process queued simulations when connection restored
6. Add timestamp and metadata to simulation records
7. Ensure storage operations don't block UI thread (use async/promises)
8. Test storage functionality in both online and offline scenarios
