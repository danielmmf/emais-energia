# PLAN-03: Implement Viability Classification and Result Display

## Objective
Implement the viability classification logic (SIM-07) and display results with appropriate visual indicators.

## Acceptance Criteria
- Service can classify results into: Alta (payback ≤ 2 anos), Média (2 < payback ≤ 4), Baixa (4 < payback ≤ 6), Estratégica (payback > 6 anos com benefícios não financeiros), Não recomendada (payback não aplicável)
- Classification considers both financial payback and strategic factors
- UI displays classification with appropriate text and color coding
- Result display includes all calculated values: custo atual, custo verde, investimento, economia anual, payback, redução de emissões
- UI shows aviso de premissas simplificadas de protótipo (TRN-01)
- Results are accessible to ReportController for report generation

## Tasks
1. Add classifyViabilidade method to ViabilityService implementing the 5-tier classification logic
2. Update calculateViabilidade method to return classification along with other results
3. Modify SimulationController to handle and display classification results
4. Create result display template with cards for each metric (RES-01)
5. Implement color-coding and text labels for each classification tier
6. Add disclaimer about simplified prototype assumptions (TRN-01)
7. Ensure result data is stored in service for ReportController access
