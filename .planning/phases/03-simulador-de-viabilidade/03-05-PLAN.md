# PLAN-05: Connect Simulation Results to Report Controller

## Objective
Ensure the ReportController can access the latest simulation results for report generation (RES-01, RES-02, RPT-01).

## Acceptance Criteria
- ReportController can access current simulation results from ViabilityService
- Report view displays all key indicators in clear cards
- System generates coherent textual recommendation based on results
- User can generate complete executory report on screen including assumptions and next steps
- Report shows explicit simplified prototype assumptions (TRN-01)
- Report functionality works regardless of storage backend (Firebase/localStorage)

## Tasks
1. Add getter method to ViabilityService for current simulation result
2. Inject ViabilityService into ReportController
3. Implement ReportController to fetch and format simulation data
4. Create/update report view with cards for each key indicator (RES-01)
5. Implement recommendation text generation based on classification and results (RES-02)
6. Create executory report view with summary, indicators, assumptions, and next steps (RPT-01)
7. Ensure TRN-01 disclaimer appears in report
8. Test report generation with various simulation outcomes
