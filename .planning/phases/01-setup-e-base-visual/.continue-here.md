# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: Telegram polling conflict — quando outro processo chama `getUpdates` em paralelo com o worker principal, a API retorna 409 e interrompe a estabilidade do bot. — mitigacao estrutural: nao usar `getUpdates` fora do worker; notificadores devem ler chats de cache/estado local.
- [ ] CONSTRAINT: `gsd-sdk query commit` nao deve ser usado para ajuda — chamada com `--help` gerou commit real inesperado. — mitigacao estrutural: usar docs/workflow para help e reservar `query commit` apenas para commits intencionais.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| polling-paralelo-telegram | `telegram_notify` e worker consumindo `getUpdates` ao mesmo tempo gerou erro 409 no bot | blocking | Centralizar `getUpdates` no worker e usar cache de chats em scripts auxiliares |
| query-commit-indevido | `gsd-sdk query commit --help` produziu commit acidental de arquivos `.planning` | blocking | Nunca usar `query commit` para help/teste; consultar workflow/skill e usar git commit/manual controlado |
