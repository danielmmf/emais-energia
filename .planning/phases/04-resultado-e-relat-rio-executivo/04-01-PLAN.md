# PLAN-04: Implementar componentes de cards de resultado

## Objective
Implementar componentes de cards de resultado para exibir os indicadores-chave da simulação de forma clara e visual.

## Requirements Addressed
- RES-01: Usuário visualiza todos os indicadores-chave em cards claros

## Tasks
1. Criar componente AngularJS para card de resultado genérico
2. Implementar formatação de valores (moeda, porcentagem, anos)
3. Criar container para exibir múltiplos cards em layout responsivo
4. Integrar com ViabilityService para receber dados de simulação
5. Implementar codificação de cores baseado na classificação de viabilidade
6. Adicionar estados de carregamento e tratamento de erros
7. Garantir responsividade para diferentes tamanhos de tela
8. Testar com diferentes cenários de simulação

## Acceptance Criteria
- [ ] Cards exibem custo atual anual, custo verde anual, investimento, economia anual, payback, redução de emissões e classificação
- [ ] Valores são formatados corretamente (R$ para moeda, % para porcentagem, anos com 2 casas decimais)
- [ ] Classification é exibida com cor adequada (Alta=verde, Média=amarelo, Baixa=laranja, Estratégica=azul, Não recomendada=vermelho)
- [ ] Layout é responsivo e funciona em desktop e tablet
- [ ] Estados de carregamento são mostrados durante processamento
- [ ] Componentes são reutilizáveis e seguem padrões existentes do aplicativo

## Dependencies
- ViabilityService (para obter dados de simulação)
- ReportController (para orquestrar exibição)
- CSS existente do aplicativo

## Estimated Effort
2 dias de desenvolvimento
