# PLAN-04: Implementar relatório executivo em tela com estrutura de apresentação

## Objective
Implementar relatório executivo completo em tela que reúne todos os indicadores, premissas e próximos passos em uma estrutura de apresentação clara para reuniões de decisão.

## Requirements Addressed
- RPT-01: Usuário gera relatório executivo em tela com resumo, indicadores, premissas e próximos passos
- TRN-01: Interface exibe aviso de premissas simplificadas de protótipo em pontos de cálculo e relatório

## Tasks
1. Criar tela/visualização dedicada para o relatório executivo
2. Estruturar layout com seções claras: cabeçalho, métricas, resumo, premissas, próximos passos
3. Integrar com ReportController para receber dados formatados do relatório
4. Exibir todos os indicadores-chave em formato de cards ou tabelas destacadas
5. Incluir seção explícita de premissas simplificadas de protótipo (TRN-01)
6. Garantir que o relatório seja visualmente atraente e fácil de ler
7. Implementar funcionalidade de impressão/visualização em tela completa
8. Garantir responsividade para diferentes tamanhos de tela
9. Testar com diferentes cenários de simulação para garantir coerência

## Acceptance Criteria
- [ ] Relatório exibe todos os indicadores-chave: custo atual anual, custo verde anual, investimento, economia anual, payback, redução de emissões e classificação
- [ ] Relatório inclui seção de resumo textual coerente com os resultados
- [ ] Relatório inclui seção de próximos passos acionáveis baseada na classificação
- [ ] Premissas simplificadas de protótipo são exibidas explicitamente (TRN-01)
- [ ] Layout é organizado e segue hierarquia visual clara para leitura
- [ ] Relatório é responsivo e funciona em desktop e tablet
- [ ] Visualização em tela adequada para apresentação em reuniões
- [ ] Fontes e cores seguem padrões existentes do aplicativo para consistência

## Dependencies
- ReportController (para obter dados estruturados do relatório)
- ViabilityService (fonte original dos dados de simulação)
- CSS existente do aplicativo
- ReportService (para estrutura de dados do relatório, se aplicável)

## Estimated Effort
2 dias de desenvolvimento
